# ADR-0027: Zero-Trust Integrity とリモート証拠のバインド

## ステータス
Accepted

## コンテキスト
これまでの投稿プロセスでは、ローカルの成果物とリモートの現実（YouTube）を決定論的に紐付ける形式的なメカニズムが欠けていました。これにより、リモートの状態を検証せずに成功と見なす「Bounded Honesty（限界ある誠実さ）」の欠如が発生するリスクがありました。また、メタデータの修正が生成ソース（JSONセグメント）と乖離する「監査ミス」も観測されました。

## 決定事項
1. **リモート証拠（Remote Proof）の必須バインド**: すべての投稿において、`UPLOAD.json` および `CONTRACT.json` に `remote_proof`（YouTube URL等）を記録することを義務付け、ローカル生産とリモート公開のループを閉じます。
2. **決定論的メタデータ派生**: メタデータ（特にタイムスタンプ/チャプター）は、静的な想定ではなく、生産成果物（`*.json` セグメント）から直接、決定論的に派生させなければなりません。
3. **状態機械の厳格化**: `ProductionState` を分解し、`LOCAL_FAIL`、`UPLOAD_CONFIRMED`、`REMOTE_UNVERIFIED` を明確に分離して管理します。
4. **ゼロ・ファット認証管理**: チャンネル（Yawa Archive 等）ごとの認証情報は、アカウント固有の `.env` ファイルで明示的に管理し、チャンネル間の汚染を防止します。

## 影響
- システムは、アセットが実際に目的地に到達したかどうかを機械的に検証可能になります。
- 実測データに依拠することで、手動によるメタデータ修正の必要性を最小限に抑えます。
- リモート状態の物理的証拠を要求することで、「監査結果の誤解釈（Audit Consumer Hallucination）」に対する耐性が向上します。
