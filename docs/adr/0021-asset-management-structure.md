# ADR-0021: 通し番号ネストによるアセット管理への移行

## ステータス
Accepted (Supercedes previous version)

## コンテキスト
フラットな通し番号管理では、アセット数が増加した際にプロジェクトやセマンティックなまとまりを把握しづらくなる懸念がありました。これを解消するため、通し番号を冠したディレクトリによる「階層的な通し番号管理」へ移行します。

## 決定事項
1. **通し番号ネストの必須化**: `/assets` 直下には通し番号（3桁以上）を冠したディレクトリのみを配置する（例: `000captions/`, `001amaamaoneesan/`）。
2. **種別分類の禁止（継続）**: ディレクトリ名は「用途」や「プロジェクト名」に基づき、`images/` などの技術的種別による分類は行わない。
3. **ファイル命名**: ディレクトリ内のファイルも原則として通し番号を付与する。
4. **追記型管理**: 番号はインクリメンタルに割り当て、一度割り当てたディレクトリ番号は変更しない。

## 影響
- アセットが意味のある単位（セッションやプロジェクト）でグループ化され、管理性が向上します。
- 通し番号による順序保証を維持しつつ、ディレクトリレベルでのコンテキストの付与が可能になります。
- `/assets` 直下の肥大化を防ぎ、ブラウジングを容易にします。
