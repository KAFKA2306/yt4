# ADR-0029: YouTube 公開設定の「原則 Public」への変更と型安全性の強化

## ステータス
Accepted

## コンテキスト
これまでの YouTube 投稿システムでは、公開設定（visibility）のデフォルト値が `unlisted`（限定公開）となっていました。また、呼び出し側で指定を忘れた場合もサイレントに限定公開として処理されていました。
しかし、本プロジェクト（yt4）の運用方針として「生成された動画は原則として即座に一般公開（public）する」ことが求められています。
設定の漏れや誤りを防ぎ、運用の透明性を確保するため、デフォルト値の変更と型システムによるガードを導入します。

## 決定事項
1. **デフォルトの公開化**: `Publisher.publish` および `Publisher.updateVideo` において、`visibility` が指定されない場合のデフォルト値を `public` に変更しました。
2. **型定義の導入**: `YouTubeVisibility` 型 (`"public" | "unlisted" | "private"`) を定義し、文字列の直接指定（Magic String）を排除しました。
3. **明示的な指定の推奨**: `Orchestrator` による自動投稿フローにおいて、デフォルト値に依存せず `visibility: "public"` を明示的に渡すように実装を修正しました。
4. **監査システム（Audit System）の強化**: `verifyContract` における `remote_proof.visibility` の検証を追加するとともに、`audit_cli.ts` に `--live` フラグを導入し、YouTube API を通じたリアルタイムの公開設定検証を可能にしました。
5. **オプトアウト構造の維持**: 必要に応じて `unlisted` や `private` を指定することは引き続き可能ですが、それは意図的な選択（Opt-in）である必要があります。

## 影響
- **公開漏れの防止**: パブリッシュフローを実行した際に、意図せず限定公開のまま放置されるリスクが解消されます。
- **型安全性の向上**: タイポによる実行時エラーや、意図しない公開設定の適用をコンパイル時に検知可能になります。
- **運用の簡素化**: ほとんどのケースで `public` を期待しているため、余分な設定記述を減らしつつ安全性を確保できます。
