# ADR-0022: 完全データ駆動型エンジンへの移行とsrc/の純粋化

## ステータス
Accepted

## コンテキスト
これまで `src/` 配下のコード（特に `Orchestrator.ts` や `index.ts`）に、シチュエーション固有のディレクトリパス、画像パス、ボイスID、キャラクター設定などのパラメータがハードコードされていました。これにより、新しいシチュエーションを追加するたびにソースコードの修正が必要となり、エンジンの汎用性と保守性が損なわれていました。

## 決定事項
1. **src/ 内のハードコード完全禁止**: ソースコード内には、特定のプロジェクトやアセットに依存する文字列（パス、ID、設定値）を一切記述しない。
2. **アセット駆動型設定 (Asset-Driven Config)**: 各アセットディレクトリ内に `0000_config.json` を配置し、以下のすべてのパラメータをそこで管理する。
   - `identity`: キャラクターID、名前、音声キャプション、パラメータ上書き。
   - `script_path`: シナリオファイル（JSON/MD）への相対パス。
   - `image_path`: 背景画像等への相対パス。
3. **ディレクトリの集約**: 入力（設定・素材）と出力（生成されたトランスクリプト・動画）を単一のアセットディレクトリ（例: `/assets/008_yandere_maid_management/`）の直下に集約する。サブディレクトリの作成は禁止する。
4. **エンジンの汎用化**: `index.ts` は引数として渡されたアセットディレクトリを読み込むだけの軽量なランナーとし、`Orchestrator` は渡された `config` に基づいて動的に動作する純粋なエンジンとして機能させる。

## 影響
- **拡張性**: `src/` を一切変更することなく、新しいディレクトリと設定ファイルを追加するだけで新規プロジェクトを実行可能になります。
- **分離**: ロジック（エンジン）とデータ（アセット・設定）が完全に分離され、システムの複雑さが軽減されます。
- **再現性**: アセットディレクトリをコピーするだけで、そのプロジェクトの全設定と生成履歴を保持・再現できるようになります。
