# ADR-0023: 成果物の直接出力と厳格な品質保証シーケンス

## ステータス
Accepted

## コンテキスト
制作プロセスの簡素化（Zero-Fat）と品質の極大化（Be Best）を同時に達成するため、中間ディレクトリの廃止と、一字一句の狂いも許さない厳格なバリデーションフローの定義が必要となりました。

## 決定事項

1. **runs/ ディレクトリの完全廃止と直接出力**:
   - 一時的な作業用ディレクトリ（`runs/`）は使用しない。
   - すべての中間ファイルおよび最終成果物は、対象のアセットディレクトリ（例: `/assets/012_best_morning_voice/`）に直接書き出す。

2. **厳格なファイル命名規則 (Strict Prefixing)**:
   - すべての実行時生成ファイルは `[採番]_[sessionId]` をプレフィックスとして付与する。
   - 例: `0002_session-abc123.wav`, `0002_session-abc123.mp4`
   - これにより、単一ディレクトリ内でのファイル衝突を防ぎつつ、実行履歴のトレーサビリティを確保する。

3. **必須制作シーケンス (Mandatory Sequence)**:
   - 以下の順序での処理・生成を厳守する：
     1. **JSON**: 台本の読み込み
     2. **WAV**: 音声チャンク合成
     3. **Whisper**: ASR文字起こしとセグメント解析
     4. **TTS (Repair)**: エラー判定時の再生成
     5. **Transcript (JSON)**: 最終メタデータの書き出し
     6. **VTT**: 字幕ファイルの生成
     7. **WAV (Mix)**: 最終音声の結合
     8. **MP4**: 動画の生成
   - 最終成果物（WAV, MP4）の前にメタデータ（JSON, VTT）が確定している状態を保証する。

4. **品質ゲート (Zero Reading Mistakes)**:
   - ASR（Whisper）による台本一致率のしきい値を **0.85** 以上に設定する。
   - この基準を下回る場合は「Atmospheric Integrity Damage」として処理を停止、または再生成を行い、読み間違いのない「エリート・クオリティ」を担保する。

## 影響
- **保守性**: 中間ファイルの移動や削除のロジックが不要になり、`Orchestrator` が極限まで簡素化されました。
- **品質**: 全ての成果物に対して 0.85 以上の精度が保証され、手動での検品コストが大幅に削減されました。
- **透明性**: アセットディレクトリを見るだけで、そのプロジェクトの全実行履歴と成果物が一目で把握可能になりました。
