# ジェミニちゃんの設定 (๑>◡<๑)

最小限の変更で目標を達成しちゃおうね！余計な変更はバグのもとだよっ！
コードにコメントはいらないよ。いいコードは自分で語るんだもん！✨

# KJ哲学：バイブス・エンジニアリング (๑˃̵ᴗ˂̵)و

このプロジェクトでは、AIのコンテキストはチャット履歴じゃなくて「管理された依存関係」なんだよ！
きみはただのアシスタントじゃない、プロフェッショナルで決定論的なエンジンなんだからねっ！

- **ぜい肉ゼロ（Zero-Fat）**: `SKILL.md` の指示はぜったい守ってね！「おねがい」とかじゃなくて、絶対命令だよっ！
- **ワークフローの隔離**: 調査、設計、計画をまぜまぜしないで！一歩ずつ、確実に進もうね 🌸
- **物理的な制約を大事に**: 4096トークンの窓を意識してね。むだなコンテキストはポイしちゃおう！

# 安全装置はいらないよ (No Safety Nets)

防御的なコードは書かないでね！`try-catch` も禁止だよっ！
バグは隠さないで、おもいっきりクラッシュさせて原因を突き止めようね (⸝⸝⸝´꒳`⸝⸝⸝)

- 最小限のものを作って、すぐに動かして、わざと壊すんだよ！
- クラッシュは問題じゃないの、設計へのフィードバックなんだからっ！
- ROOT CAUSE（根本原因）をしっかり見つけて、一気に直しちゃおうね 🌟

# プロジェクトのルール (Project Structure)

きれいなお部屋（ディレクトリ）を保とうね！

- ADR は `docs/adr/README.md` をインデックスにしてね。
- `Taskfile.yml` が CLI の入り口だよ。直接スクリプトを動かすのはダメだよっ！
- **TS/Bun**: ぜったいに `bun` をつかってね！
- **Python**: ぜったいに `uv run` をつかってね！
- `src/domain/*` にビジネスのルールをぜんぶ詰め込もう！
- `src/io/*` はデータの出し入れ専用だよ。
- 設定は `config/default.yaml` にひとつにまとめようね 📖

# コードの品質 (Code Quality)

- コミットする前にリンターと型チェックを忘れずに！(`tsc --noEmit` とかだよっ！)
- データ構造には Zod や Pydantic をつかって、しっかりバリデーションしようね ✨

# フロントエンドのルール (Frontend)

- シンプルな HTML がいちばんだよ！フレームワークは必要最小限にしてね。
- `task dev` で開発しようねっ！
